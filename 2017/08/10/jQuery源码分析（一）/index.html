jQuery源码分析（一）

Posted on 2017-08-10

jQuery源码分析之基本框架搭建

这一篇先整体介绍jQuery结构的内容，接下来的每一篇会对每一部分做详细介绍，以帮助大家更好理解。

1.jQuery、$理解
首先，在jQuery最后面，有这样一段代码，匿名函数自执行，那么为什么要这么做呢？

( {
if ( !noGlobal ) {
    window.jQuery = window.$ = jQuery;
}
return jQuery;
} );

好处：在匿名函数里面写的任何内容都是局部的，在外面访问不到函数里面的内容，所以jQuery把代码都写在匿名函数里，防止跟其他内容产生冲突，互不影响，也就是我们平时所说的全局变量污染。但是只把这些内容放在局部中也不行，我们还需要提供对外的接口，那么怎样在jQuery中提供对外的接口呢？jQuery中是这样做的，将提供对外的接口挂载在window下面，这样再来调用的时候其实是调用window下面的。jQuery中的源码：

window.jQuery = window.$ = jQuery;

这样就可以通过$方法或者是jQuery方法提供对外的接口了。

2.原型
原型是属于面向对象的东西，所以说jQuery就是基于面向对象的一个程序，里面内容是跟面向对象有关的。

(function(){
    //给jQuery对象添加一些方法和属性。
    jQuery.fn = jQuery.prototype = {}
})

在jQuery源码中可以看到jQuery下面函数执行完之后return的结果就是一个jQuery对象，对象可以调用方法，所以jQuery中都是方法调用。

jQuery = function( selector, context ) {

   // The jQuery object is actually just the init constructor 'enhanced'
   // Need init if jQuery is called (just allow error to be thrown if not included)
 return new jQuery.fn.init( selector, context );
}

3.继承
jQuery的继承方法，通过后续添加的代码，都可以通过extend方法把它挂在jQuery对象中，很方便的去扩展，而不是说一上来就把所有的方法都写在一起，都写在一些不利于后期代码的独立性、维护和扩展。

(function(){
    //jQuery的继承方法
    jQuery.extend = jQuery.fn.extend = function() {}
})

jQuery中扩展工具方法：jQuery中提供了两种扩展工具方法：

(function(){
    jQuery.extend( {} )
    //两种操作代码的方法：
    //第一种：通过对象调用方法
    $("div").css()
    //第二种，通过函数调用方法,扩展一些静态工具方法，可以给jQuery对象用，也可以给原生的js用。
    $.trim()
})

4.延迟管理
下面代码的实现延迟对象，对异步的统一管理。如果没有延迟对象，那么会先弹出456，隔1秒再弹出123，但是加了延迟对象，下面的代码先弹出123，因为456是在函数里面的，如果没有调用，就不会执行，所以先弹出123之后，执行dfd.resolve()，调用done()，实现的原理其实是回调函数。

(function(){
    //3386行—3477行，实现延迟对象，对异步的统一管理
    //Deferred: function( func ){}
    var dfd= $.Deferred();
    setTimeout(function(){
        alert(123);
        dfd.resolve();
    },1000)
    dfd.done(function(){
        alert(456);
    })
})

5.功能检测
support的功能检测，判断游览器版本，比如，在input中，默认的value是多少呢？可以通过support.checkOn()来判断，大部分预览器默认的是2，所以返回true，但是在老版本的游览器中，默认是空(“”)，返回false，所以这个时候如果我们想区分老版本的游览器和新版本的游览器就可以通过这个来检测，

jQuery中数据没有扩展到元素身上，数据缓存机制，防止内存泄漏，如果没有扩展到基本类型身上，而是元素身上，那么就可能出现内存泄漏的问题，

6.队列管理
queue入队管理，dequeue出队管理，那么在哪里会用到队列呢？在jQuery中，我们常用到的是运动，比如：

(function(){
    data();
    $("div").animate({left:100})
    $("div").animate({top:100})
    $("div").animate({width:100})
    //这个是分开写的，那么在jQuery怎么保证顺序呢？就是用到了队列管理，作用是先把这三个运动存到队列中，
    // 然后当一个运动完后，再调用dequeue方法，后续的代码才会继续执行。
})
